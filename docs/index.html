Pi Google Stitch Tools

@zyvra-labs/pi-stitch-tools v0.3.3

@zyvra-labs/pi-stitch-tools is the dedicated Pi (pi.dev) package for Google Stitch. Install it and you get Stitch tools inside Pi — generate, edit, and manage UI designs from text, without manually configuring MCP.

📦 GitHub 📋 npm 🎨 Google Stitch

Install

pi install npm:@zyvra-labs/pi-stitch-tools

After installing, restart Pi or run /reload.

Configure

Set your Stitch API key via any of these methods:

Environment variable: 
export STITCH_API_KEY="your-key"
# or
export GOOGLE_STITCH_API_KEY="your-key"
File: 
echo "your-key" > ~/.pi/stitch-api-key
Legacy mcp.json: if you already have Stitch in ~/.pi/agent/mcp.json, it's auto-detected.

Verify

/stitch-status

Expected output: Stitch: 14 tools loaded

Commands

Command Description
/stitch-status Show loaded tools and API status
/stitch-projects List your Stitch projects
/stitch-new-screen <id> <prompt> Generate a screen from description
/stitch-theme <id> List design systems for a project
/stitch-help Show all available Stitch commands

Available Tools

All Google Stitch tools are available under the stitch_ prefix:

  • stitch_list_projects — List your Stitch projects
  • stitch_get_project — Get project details
  • stitch_create_project — Create a new project
  • stitch_list_screens — List screens in a project
  • stitch_get_screen — Get screen details
  • stitch_generate_screen_from_text — Generate a screen from text
  • stitch_edit_screens — Edit existing screens
  • stitch_generate_variants — Generate design variants
  • stitch_create_design_system — Create a design system
  • stitch_update_design_system — Update a design system
  • stitch_apply_design_system — Apply design system to screens
  • stitch_list_design_systems — List design systems
  • stitch_upload_design_md — Upload DESIGN.md
  • stitch_create_design_system_from_design_md — Create DS from DESIGN.md

When to use this package

✅ Use @zyvra-labs/pi-stitch-tools if:

  • You want to use Google Stitch from Pi without manually configuring an MCP client
  • You need Stitch tools available as native Pi slash commands (/stitch-status, /stitch-projects)
  • You want automatic tool discovery — new Stitch tools appear without updating the package

❌ Use a generic MCP adapter instead if:

  • You need to connect many different MCP servers with full custom configuration
  • You require fine-grained control over MCP transport, auth, and tool filtering

Examples

Create a new screen:

/stitch-new-screen my-project "A SaaS analytics dashboard with cards and charts"

List your projects:

/stitch-projects

Generate from natural language:

stitch_generate_screen_from_text projectId=14082150201392761251 prompt="Login page with email, password, and social login buttons"

Create a design system:

stitch_create_design_system projectId=14082150201392761251 designSystem='{"displayName":"SaaS Dark","theme":{"colorMode":"DARK","headlineFont":"INTER","bodyFont":"INTER","roundness":"ROUND_EIGHT","customColor":"#4f46e5"}}'

Troubleshooting

Problem Solution
/stitch-status shows 0 tools Run /reload, then check your API key with
cat ~/.pi/stitch-api-key or echo $STITCH_API_KEY
Authentication failed Make sure your Stitch API key is valid and has access to Google Stitch
API key not found Set STITCH_API_KEY or GOOGLE_STITCH_API_KEY, or write key to ~/.pi/stitch-api-key
Stitch unavailable Check internet connection. The extension connects to stitch.googleapis.com/mcp

How it works

The extension connects directly to the Google Stitch MCP endpoint (stitch.googleapis.com/mcp) over HTTP. Tools are registered dynamically from the server's tools/list response — no hardcoded tool definitions, no MCP client dependency, only Node.js built-ins.
